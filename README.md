# Cloud Engineering Challenge

## Part 4 — Research Task

### 1. What is cloud computing?
Cloud computing is the delivery of computing services like servers, storage,
databases, and software over the internet. Instead of owning physical hardware,
you rent what you need from providers like AWS, Google Cloud, or Azure and only
pay for what you use.

### 2. What is DevOps?
DevOps is a way of working where development and operations teams collaborate
closely to build, test, and release software faster and more reliably. It uses
automation and continuous delivery to reduce the time between writing code and
getting it to users.

### 3. Difference between Docker and Kubernetes?
Docker packages an application and everything it needs to run into a container.
Kubernetes manages many containers at once — it handles scaling, restarting
failed containers, and spreading load across servers. Docker creates the box,
Kubernetes organizes the warehouse full of boxes.

### 4. Why are Linux skills important in cloud engineering?
Almost all cloud servers run on Linux. Knowing Linux means you can navigate
servers, write automation scripts, manage files and permissions, and troubleshoot
problems — all essential daily tasks in cloud engineering.

## Part 5 — Problem Solving

### If a cloud server becomes inaccessible, I would check:
1. Network connectivity — is the server reachable via ping or SSH?
2. Server status — is the instance actually running in the cloud console?
3. Firewall/Security rules — is port access blocked by a security group or rule?
